Code Guidelines

The bulk of our Style-guide's code consists of single-purpose classes which, as their name suggests, are classes that declare a single value for a single property. This division of concerns helps keep the code modular and easy to maintain. The main benefit of using single-purpose classes is that when an engineer will know exactly how each class affects the markup and will be able to quickly develop or alter views without affecting others.

Our style-guide follows the Atomic Design structure. The majority of the code base consists of single purpose classes which are used to create may different atoms such as buttons and inputs. These many different atoms can be further combined together to form molecules which are components that put the atoms to good use. These components should be used in development to keep the platform's design consistent.

Code Guidelines

The bulk of our styling framework's code consists of custom made components and bootstrap overrides. This division of concerns helps keep the code modular and easy to maintain. All those components take their styles from a set of predefined themes.

Our style-guide follows the Atomic Design structure. Our themes consist of maps where the key is the property name and the value is the property value. which are used to create may different atoms such as buttons and inputs. These many different atoms can be further combined together to form molecules which are components that put the atoms to good use. These components should be used in development to keep the platform's design consistent.

File Naming

All folder, file, and property names in our framework should be singular. Style files that are to be imported to the global theme should have the 'theme' subclass added.
Additionally any files related to components should also have the 'component' added.

/* Do */ @import '_card.component.theme.scss'; @import '_bootstrap.theme.scss'; /* Don't */ @import 'card.scss'; @import 'bootstrap.scss';

Theme Combination Naming

Keys in theme maps should all be in lower-kebab-case meaning that capital letters are never used and words are delimited by hyphens. And whenever we have multiple keys with the same values, all the keys should reference the very first one to avoid redundancy and improve performance.
Their intuitive design will get all developers familiar with them as quick as possible, with some special classes designed

/* Do */ color-primary: #209e91 color-secondary-gray: #e5e5e5 color-text-primary: color-primary-gray /* Don't */ ColorPrimary: #209e91 ColorSecondaryGray: #e5e5e5 ColorTextPrimary: #e5e5e5

Mixins and Directives

All component styles and overrides are applied using mixins. And all components are implemented as shadow DOM elements (directives).
Component style mixin names should begin with 'nga' followed by the component name, and end with 'theme'.
Bootstrap overrides mixin names follow the same rules except they also include a 'b'.
Finally, all custom component tags should include 'nga' at the beginning.
All names are in lower-camel-case.

/* Mixins */ @mixin nga-actions-theme() @mixin nga-card-theme() @mixin nga-b-accordion-theme() @mixin nga-b-button-group-theme() /* Directives */ <nga-card> </nga-card> <nga-search-field> </nga-search-field>

Applying The Theme

Any section of code where the activated theme(s) should be applied must be enclosed inside the nga-layout tags. This gives us more control over our styles and view encapsulation.

/* nga-layout */ <nga-layout> <nga-layout-columns> <h1> Hello World! <h1> </nga-layout-columns> </nga-layout>

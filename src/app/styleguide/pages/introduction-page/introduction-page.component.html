Introduction

With the expansion of the MonopolyKings team on the horizon, we thought ahead about our needs as a development team. More specifically we wanted to handle scalability in a graceful manner in order to ensure that our platform's quality is maintained and it's design consistent.We are building this style-guide to refine and maintain our brand identity. This style-guide consists of a library of tailored UI components, brand guidelines, brand assets, code snippets, developer guidelines and more. All of MonopolyKings' engineers should abide by these guidelines and are encouraged to constantly add their input raise new ideas for discussion.

Coherent code style and structure

We want all our engineers to write code that is understandable and easily maintainable by everyone.

Set our brand identity

We would like to create a global and unique brand identity for MonopolyKings. Make it easily transferable to all platforms.

Speed up development

Our engineers will now be able to create interfaces at lightning speed thanks to our UI library and Atomic Design structure.
